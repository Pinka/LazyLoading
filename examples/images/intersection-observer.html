Lazy Loading with Intersection Observer API

How It Works

This example demonstrates image lazy loading using the Intersection Observer API. Images will only load when they are about to enter the viewport.

Key differences from native lazy loading:

  • More control over loading behavior
  • Custom loading animations/placeholders
  • Works in browsers that don't support the native loading="lazy" attribute
  • Can be used for any element, not just images

What to Observe

In your browser's developer tools (Network tab):

  • The first image should load immediately when the page loads
  • The other images should only load when you scroll down close to them
  • Notice the timing of when each request is made
  • Observe the placeholder being replaced with the actual image

The Code

const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    // When placeholder is visible
    if (entry.isIntersecting) {
      const placeholder = entry.target;
      const imgSrc = placeholder.dataset.src;
      
      // Create new image element
      const img = new Image();
      img.src = imgSrc;
      img.alt = "Lazy loaded image";
      img.className = "lazy-image";
      
      // When image is loaded, replace placeholder
      img.onload = () => {
        placeholder.parentNode.replaceChild(img, placeholder);
      };
      
      // Stop observing this element
      observer.unobserve(placeholder);
    }
  });
}, {
  // Start loading when image is 200px away
  rootMargin: '200px 0px',
  threshold: 0.01
});

// Observe all placeholders
document.querySelectorAll('.placeholder[data-src]').forEach(placeholder => {
  observer.observe(placeholder);
});
